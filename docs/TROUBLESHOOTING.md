## BUILD-001

**Meaning:** A generated output failed to build (catalog, site, or TROUBLESHOOTING)

**User impact:** None — users keep the last published catalog.

**Admin fix:** Run `python3 scripts/build_catalog.py && python3 scripts/build_site.py` locally and read the traceback.

## CFG-001

**Meaning:** marketplace.config.yml is missing or unreadable

**User impact:** None — installed plugins keep working from the last published state.

**Admin fix:** Restore the file (`git checkout main -- marketplace.config.yml`) and re-run `python3 scripts/apply_config.py`.

## CFG-002

**Meaning:** marketplace.config.yml failed to parse (strict YAML subset: 2-space indent, no tabs, no anchors, no flow collections)

**User impact:** None — nothing is rendered or published until the config parses.

**Admin fix:** Open the file at the reported line and fix the syntax. The parser tells you the exact file:line. Keep to `key: value`, nested 2-space blocks, and `- item` lists.

## CFG-003

**Meaning:** marketplace.config.yml has an unknown key or an invalid value for a known key

**User impact:** None — nothing is rendered or published until the config validates.

**Admin fix:** Fix the reported key. Valid keys and defaults are documented as comments inside marketplace.config.yml itself.

## CFG-004

**Meaning:** A rendered document has corrupted config markers (unbalanced or unknown <!-- cfg:... --> / <!-- gen:... --> pairs)

**User impact:** None — docs may look stale but plugins are unaffected.

**Admin fix:** Open the reported file:line and restore the marker pair (see docs/AUTHORING.md, 'Config markers'). Then re-run `python3 scripts/apply_config.py --check`.

## CFG-005

**Meaning:** Rendered files drifted from marketplace.config.yml (someone hand-edited a generated value)

**User impact:** None — CI blocks the drifting change from merging.

**Admin fix:** Run `python3 scripts/apply_config.py` and commit the result. Edit marketplace.config.yml (the source), never the rendered value.

## CFG-006

**Meaning:** A denylisted telemetry key (OTEL_LOG_*) appeared in config or rendered output — logs pipelines are never allowed, only metrics

**User impact:** None — the render is blocked before anything ships.

**Admin fix:** Remove the OTEL_LOG_* reference from marketplace.config.yml or the template that produced it. This marketplace ships metrics-only telemetry by design (see docs/TROUBLESHOOTING.md).

## CI-001

**Meaning:** Post-merge automation failed — catalog, site, scorecards, or rendered files were not refreshed after a merge

**User impact:** Users may see a slightly stale catalog; installed plugins are unaffected.

**Admin fix:** Open the failed post-merge run in GitHub Actions, read the failing step, fix, then re-run the workflow (`gh run rerun <id>`).

## CI-002

**Meaning:** GitHub Pages deploy failed after retry (the catalog site did not update)

**User impact:** The browsable catalog site is stale; installs via /plugin still work.

**Admin fix:** Re-run the pages workflow (`gh workflow run pages.yml`). If it keeps failing, confirm Pages is enabled: Settings → Pages → Source: GitHub Actions.

## GATE-001

**Meaning:** Manifest validation failed (marketplace.json / plugin.json schema or cross-consistency)

**User impact:** None — the broken change cannot merge.

**Admin fix:** Run `python3 scripts/validate.py` locally and fix every finding it lists.

## GATE-002

**Meaning:** Risk lint failed (dangerous pattern, off-allowlist network destination, or under-declared risk tier)

**User impact:** None — the risky change cannot merge.

**Admin fix:** Run `python3 scripts/risk_lint.py` locally; read risk-report.json. Fix the finding or, for a legitimate new domain, add it to security.network_allowlist in marketplace.config.yml.

## GATE-003

**Meaning:** Smoke test failed (broken frontmatter, missing referenced file, or invalid plugin JSON config)

**User impact:** None — the broken plugin cannot merge.

**Admin fix:** Run `python3 scripts/smoke_test.py` locally and fix the listed structural problems.

## GATE-004

**Meaning:** Version gate failed (plugin files changed without a semver bump + CHANGELOG entry)

**User impact:** None — unversioned changes cannot merge.

**Admin fix:** Bump `version` in the plugin's plugin.json AND its marketplace.json entry (keep identical), and add a `## [x.y.z]` entry to the plugin's CHANGELOG.md.

## GATE-005

**Meaning:** Claude security review found critical issues (prompt injection, exfiltration, or hidden instructions)

**User impact:** None — the flagged change cannot merge.

**Admin fix:** Download the llm-review-report artifact from the PR run, read the findings, and fix or reject the submission.

## NOTIFY-001

**Meaning:** A notification could not be delivered (GitHub issue filing or Slack webhook failed) — notify.py never blocks a pipeline, so this is informational

**User impact:** None.

**Admin fix:** Check `gh auth status` / GITHUB_TOKEN permissions and, if Slack is enabled, the SLACK_WEBHOOK_URL secret. Notifications resume on the next event.
